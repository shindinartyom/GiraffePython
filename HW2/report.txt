Я использовал Redis.

Запустить sync агента:
uvicorn sync:app --reload
Запустить async агента:
uvicorn async:app --reload
python async.py
Протестировать агента (в отдельном терминале):
python test.py

В синхронной версии время выполнения растет окололинейно.
В асинхронной версии время ответа API остается стабильно низким (миллисекунды), так как нагрузка перекладывается на воркеров.
Разница стала очевидной уже на 10 запросах.

API с брокером отвечает быстрее потому что теперь он сам не выполняет задачу. Его единственная обязанность — зафиксировать факт поступления запроса, пингануть Redis и вернуть статус. Вся работа теперь на вокере, который отделен от веб-сервиса.

Минусы async:
- Теперь API и задача разделены и приходят в разное время, соответственно нужна дополнительная логика для получения результата
- Дополнительно нужно мониторить воркер процессы
- Если воркер упадет, API продолжит принимать задачи в очередь, и пользователь не сразу узнает о проблеме без дополнительной логики проверки статуса самой задачи
